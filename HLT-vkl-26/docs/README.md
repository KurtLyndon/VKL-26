# Docs Index

Thư mục `docs` đã được tách theo 3 nhóm đối tượng để dễ quản lý và tránh lẫn ngữ cảnh:

## 1. User

Thư mục: `docs/user`

Dành cho:

- người vận hành
- người dùng hệ thống
- người cần xem hướng dẫn sử dụng ở mức thao tác

## 2. Developer

Thư mục: `docs/developer`

Dành cho:

- lập trình viên
- người viết script cho agent
- người chỉnh schema, migration, contract kỹ thuật

## 3. Cath

Thư mục: `docs/cath`

Dành riêng cho:

- ghi nhớ phiên làm việc tiếp theo
- chỉ dẫn ngắn gọn để Codex/Cath đọc trước khi bắt đầu
- các ghi chú nội bộ không nên trộn chung vào tài liệu user/developer

## Quy ước đọc tài liệu

- User: ưu tiên `docs/user`
- Developer: ưu tiên `docs/developer`
- Cath: mặc định chỉ đọc `docs/cath` trước; chỉ mở thêm tài liệu ở `docs/developer` khi tác vụ yêu cầu
